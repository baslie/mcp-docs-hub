## RULE: Запрет на создание файла `nul`

**КРИТИЧЕСКИ ВАЖНО:** В Windows `nul` — зарезервированное имя устройства (аналог `/dev/null`). Создание файла с таким именем приводит к проблемам.

### Запрещённые действия

НИКОГДА не выполняй команды, создающие файл `nul`:

```bash
# ЗАПРЕЩЕНО:
> nul
echo "" > nul
Write-Output > nul
```

### Правильные альтернативы

Для подавления вывода используй:

```bash
# Windows CMD:
command > NUL 2>&1

# PowerShell:
command | Out-Null
command > $null

# Git Bash:
command > /dev/null 2>&1
```

### Если файл `nul` уже создан

Немедленно удали его командой Git Bash:

```bash
rm "./nul"
```

Стандартные команды Windows (`del`, `Remove-Item`) не могут удалить этот файл из-за зарезервированного имени.

### Другие зарезервированные имена Windows

Аналогичные ограничения действуют для: `con`, `prn`, `aux`, `com1`-`com9`, `lpt1`-`lpt9`.
