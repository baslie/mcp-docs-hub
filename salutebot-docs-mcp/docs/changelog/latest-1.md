# История обновлений
В этом разделе вы найдете историю обновлений и изменений документации SaluteBot.
## Сентябрь
  * Рассказали о новом отчете **Воронки** , который позволяет узнать поведение клиента на каждом шаге [сценария](https://developers.sber.ru/docs/ru/salutebot/statistic-reports/reports?tool=code#voronki) и проанализировать качество работы чат-бота.

## Август
  * Описали новый блок [AI-ответ из базы знаний](../scenario/blocks/ai-answer.md): он позволяет пользователям получить ответ, сгенерированный нейросетью GigaChat на основе информации из загруженных документов.
  * Теперь при создании чат-ботов с помощью базы знаний можно [загружать документы по URL](../bot-steps/bot-creation-by-docs.md#sozdanie-bazy-znaniy).
  * Добавили полезную информацию об [отчетах](../statistic-reports/reports.md), позволяющих анализировать работу чат-бота: **Диалоги** , **Популярные интенты** и **Активность**.

## Июль
  * Рассказали про возможность подключать чат-бот напрямую к [Telegram](../bot-channel/telegram.md), без использования дополнительных чат-платформ.

## Июнь
  * Описали способы интеграции чат-ботов с платформами [EDNA](../edna-integration.md) и [Webim](../webim-integration.md).

## Апрель
  * Теперь можно подключить сервис как для физического, так и для юридического лица с любой страницы документации. Подключите свой чат-бот сразу.
  * Добавили возможность создавать SaluteBot [используя имеющиеся документы](../bot-steps/bot-creation-by-docs.md) – создайте ваш чат-бот не погружаясь в разработку.

## Январь
  * Создать чат-бот стало еще проще: используйте [шаблон Бот-консультант GigaChat](../bot-steps/wizard-creation.md). В созданном шаблоне сценария ответы будут генерироваться автоматически.
